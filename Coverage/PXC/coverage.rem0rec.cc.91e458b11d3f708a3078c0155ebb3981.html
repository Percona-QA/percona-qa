143 /* CANONICAL COORDINATES. A record can be seen as a single
144 string of 'characters' in the following way: catenate the bytes
145 in each field, in the order of fields. An SQL-null field
146 is taken to be an empty sequence of bytes. Then after
147 the position of each field insert in the string
148 the 'character' <FIELD-END>, except that after an SQL-null field
149 insert <NULL-FIELD-END>. Now the ordinal position of each
150 byte in this canonical string is its canonical coordinate.
151 So, for the record ("AA", SQL-NULL, "BB", ""), the canonical
152 string is "AA<FIELD_END><NULL-FIELD-END>BB<FIELD-END><FIELD-END>".
153 We identify prefixes (= initial segments) of a record
154 with prefixes of the canonical string. The canonical
155 length of the prefix is the length of the corresponding
156 prefix of the canonical string. The canonical length of
157 a record is the length of its canonical string.
158
159 For example, the maximal common prefix of records
160 ("AA", SQL-NULL, "BB", "C") and ("AA", SQL-NULL, "B", "C")
161 is "AA<FIELD-END><NULL-FIELD-END>B", and its canonical
162 length is 5.
163
164 A complete-field prefix of a record is a prefix which ends at the
165 end of some field (containing also <FIELD-END>).
166 A record is a complete-field prefix of another record, if
167 the corresponding canonical strings have the same property. */
