39 /* The format of compressed pages is as follows.
40
41 The header and trailer of the uncompressed pages, excluding the page
42 directory in the trailer, are copied as is to the header and trailer
43 of the compressed page.
44
45 At the end of the compressed page, there is a dense page directory
46 pointing to every user record contained on the page, including deleted
47 records on the free list. The dense directory is indexed in the
48 collation order, i.e., in the order in which the record list is
49 linked on the uncompressed page. The infimum and supremum records are
50 excluded. The two most significant bits of the entries are allocated
51 for the delete-mark and an n_owned flag indicating the last record in
52 a chain of records pointed to from the sparse page directory on the
53 uncompressed page.
54
55 The data between PAGE_ZIP_START and the last page directory entry will
56 be written in compressed format, starting at offset PAGE_DATA.
57 Infimum and supremum records are not stored. We exclude the
58 REC_N_NEW_EXTRA_BYTES in every record header. These can be recovered
59 from the dense page directory stored at the end of the compressed
60 page.
61
62 The fields node_ptr (in non-leaf B-tree nodes; level>0), trx_id and
63 roll_ptr (in leaf B-tree nodes; level=0), and BLOB pointers of
64 externally stored columns are stored separately, in ascending order of
65 heap_no and column index, starting backwards from the dense page
66 directory.
67
68 The compressed data stream may be followed by a modification log
69 covering the compressed portion of the page, as follows.
70
71 MODIFICATION LOG ENTRY FORMAT
72 - write record:
73 - (heap_no - 1) << 1 (1..2 bytes)
74 - extra bytes backwards
75 - data bytes
76 - clear record:
77 - (heap_no - 1) << 1 | 1 (1..2 bytes)
78
79 The integer values are stored in a variable-length format:
80 - 0xxxxxxx: 0..127
81 - 1xxxxxxx xxxxxxxx: 0..32767
82
83 The end of the modification log is marked by a 0 byte.
84
85 In summary, the compressed page looks like this:
86
87 (1) Uncompressed page header (PAGE_DATA bytes)
88 (2) Compressed index information
89 (3) Compressed page data
90 (4) Page modification log (page_zip->m_start..page_zip->m_end)
91 (5) Empty zero-filled space
92 (6) BLOB pointers (on leaf pages)
93 - BTR_EXTERN_FIELD_REF_SIZE for each externally stored column
94 - in descending collation order
95 (7) Uncompressed columns of user records, n_dense * uncompressed_size bytes,
96 - indexed by heap_no
97 - DATA_TRX_ID_LEN + DATA_ROLL_PTR_LEN for leaf pages of clustered indexes
98 - REC_NODE_PTR_SIZE for non-leaf pages
99 - 0 otherwise
100 (8) dense page directory, stored backwards
101 - n_dense = n_heap - 2
102 - existing records in ascending collation order
103 - deleted records (free list) in link order
104 */
