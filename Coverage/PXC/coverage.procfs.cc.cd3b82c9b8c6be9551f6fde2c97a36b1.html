15 /**
16 @file
17 The ProcFS plugin is intended to provide access to Linux performance counters
18 by running SQL queries against a Percona Server 8.0 (or other MySQL 8.0
19 compatible variant). This is useful for Cloud and MySQL-as-service
20 installations.
21
22 The plugin reads files from /proc and /sys filesystems and returns the files
23 name and content as is within individual rows of a new view
24 INFORMATION_SCHEMA.PROCFS.  The schema definition of this view is:
25
26 CREATE TEMPORARY TABLE `PROCFS` (
27 `FILE` varchar(1024) NOT NULL DEFAULT '',
28 `CONTENTS` longtext NOT NULL
29 ) ENGINE=InnoDB DEFAULT CHARSET=utf8;
30  
31
32 Access to /proc and /sys dfiles and directories is limited by the system
33 variable procfs_files_spec. This variable is a global, read only variable,
34 meaning that it may only be set either via the mysqld command line or within
35 my.cnf and is not changeable at run time.  This is for security purposes to
36 prevent a compromised account from giving itself greater access to the /proc
37 or /sys filesystems.  Any files that are not included within the
38 procfs_files_spec are ignored and considered to not exist.  You can write file
39 access patterns in glob(7) style:  /sys/block/sd[a-z]/stat;/proc/version*
40
41 The default value for procfs_files_spec is : /proc/cpuinfo;/proc/irq//;
42 /proc/loadavg;/proc/net/dev;/proc/net/sockstat;/proc/net/sockstat_rhe4;
43 /proc/net/tcpstat;/proc/self/net/netstat;/proc/self/stat;/proc/self/io;
44 /proc/self/numa_maps/proc/softirqs;/proc/spl/kstat/zfs/arcstats;/proc/stat;
45 /proc/sys/fs/file-nr;/proc/version;/proc/vmstat
46
47 To install this plugin, copy the procfs.so file to the mysql plugin
48 installation directory and execute "INSTALL PLUGIN procfs SONAME 'procfs.so';"
49
50 Access to the INFORMATION_SCHEMA.PROCFS is limited to users with the
51 ACCESS_PROCFS dynamic privilege.  Upon plugin startup, this new dynamic
52 privilege is registered with the server.  After the plugin has been installed,
53 grant a user access to the INFORMATION_SCHEMA.PROCFS view by executing:
54 "GRANT ACCESS_PROCFS ON *. TO 'user'@'host';".
55
56 Authorized users can now obtain information from individual files by
57 specifying the exact file name within a WHERE clause. Example:
58 "SELECT * FROM INFORMATION_SCHEMA.PROCFS WHERE FILE = '/proc/version';".
59 Specifying specific files is critical to limiting the impact of the plugin to
60 the servers performance.  Limited file globbing is supported by the LIKE
61 expression.  Failure to limit the set of files scanned via a WHERE clause
62 through equality, LIKE or IN qualifiers can lead to lengthy query response
63 times and high load and memory usage on the server as all files that match the
64 procfs_files_spec must be opened, read, held in memory, and finally returned
65 to the client.
66
67 Some basic metrics are provided by status variables:
68
69 Name Description
70 -----------------------------------------------------------------------------
71 procfs_access_violations number of attempted queries by users without the
72 ACCESS_PROCFS privilege
73 procfs_queries number of queries made against the procfs view
74 procfs_files_read number of files read to provide content
75 procfs_bytes_read number of bytes read to provide content
76
77
78 Credit goes to Nickolay Ihalainen (nickolay.ihalainen@percona.com) for the
79 original implementation on which this work is based. Thank you Nickolay!
80 */
