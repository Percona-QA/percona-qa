156 /*
157 Helper methods to deal with thread local storage.
158 The purpose of these methods is to hide the details of thread
159 local storage handling when operating with wsrep storage access
160 and streaming applier THDs
161
162 With one-thread-per-connection thread handling thread specific
163 variables are allocated when the thread is started and deallocated
164 before thread exits (my_thread_init(), my_thread_end()). However,
165 with pool-of-threads thread handling new thread specific variables
166 are allocated for each THD separately (see threadpool_add_connection()),
167 and the variables in thread local storage are assigned from
168 currently active thread (see thread_attach()). This must be taken into
169 account when storing/resetting thread local storage and when creating
170 streaming applier THDs.
171 */
172

174 Create new variables for thread local storage. With

178 the thread local storage before this call.
179

181 */
