44 @page PAGE_TABLE_ACCESS_SERVICE Table Access service
45
46 The TABLE ACCESS service allows components to read and write
47 to MySQL tables owned by the component.
48
49 The table access service is meant to be used to operate on
50 "well known" tables, with a known name and structure.
51 It is not meant to access arbitrary tables in a generic way.
52
53 The overall service consists of the following parts:
54 - The table access factory (@ref s_mysql_table_access_factory_v1)
55 - The table access (@ref s_mysql_table_access_v1)
56 - The table access index (@ref s_mysql_table_access_index_v1)
57 - The table access scan (@ref s_mysql_table_access_scan_v1)
58 - The table access update (@ref s_mysql_table_access_update_v1)
59
60 To manipulate table columns in a type safe manner,
61 and to decouple data types from the general service,
62 each data type supported is manipulated using the following parts:
63 - The field access for integer types (@ref s_mysql_field_integer_access_v1)
64 - The field access for varchar types (@ref s_mysql_field_varchar_access_v1)
65
66 In addition, to manipulate fields in a generic way,
67 each field can be read from and written to using 'any':
68 - The field access for any types (@ref s_mysql_field_any_access_v1)
69
70 All these parts are related, and share the @c Table_access type.
71
72 To read from, or write to, a MySQL table from a component,
73 the component code must, in that order:
74 - create a table access object, using the factory,
75 - populate the table access object, with all the tables to open,
76 - call the begin operation,
77 - check that the table DDL for each table corresponds to the component
78 expectations.
79
80 At this point, the tables are ready for use.
81 Supported operations are:
82 - full table scan,
83 - index open,
84 - index scans,
85 - index fetch,
86 - insert, update or delete on a row.
87
88 For table access sessions that update data, the session can be committed.
89
90 To complete the session, either commit or rollback,
91 then destroy the session object.
92
93 The table access service only supports basic DML operations (no DDL).
94
95 Please note that, in all the code examples below:
96 - the variables "*_srv" points to the proper service part,
97 without details,
98 - proper error handling is missing.
99
100 This is deliberate, to have legible code examples.
101
