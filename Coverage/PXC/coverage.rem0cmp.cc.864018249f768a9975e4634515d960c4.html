49
50 The records are put into alphabetical order in the following
51 way: let F be the first field where two records disagree.
52 If there is a character in some position n where the
53 records disagree, the order is determined by comparison of
54 the characters at position n, possibly after
55 collating transformation. If there is no such character,

57 if the data type of the fields is paddable,
58 shorter field is padded with a padding character. If the
59 data type is not paddable, longer field is considered greater.
60 Finally, the SQL null is bigger than any other value.
61
