6 * This file implements a 64-bit Murmur-2 hash UDF (user-defined function) for
7 * MySQL. The function accepts any number of arguments and returns a 64-bit
8 * unsigned integer. MySQL actually interprets the result as a signed integer,
9 * but you should ignore that. I chose not to return the number as a
10 * hexadecimal string because using an integer makes it possible to use it
11 * efficiently with BIT_XOR().
12 *
13 * The function never returns NULL, even when you give it NULL arguments.
14 *
15 * To compile and install, execute the following commands. The function name
16 * murmur_hash in the mysql command is case-sensitive! (Of course, when you
17 * actually call the function, it is case-insensitive just like any other SQL
18 * function).
19 *
20 * g++ -fPIC -Wall -I/usr/include/mysql -shared -o murmur_udf.so murmur_udf.cc
21 * cp murmur_udf.so /lib
22 * mysql mysql -e "CREATE FUNCTION murmur_hash RETURNS INTEGER SONAME
23 * 'murmur_udf.so'"
24 *
25 * Once installed successfully, you should be able to call the function. Here's
26 * a faster alternative to MD5 hashing, with the added ability to hash multiple
27 * arguments in a single call:
28 *
29 * mysql> SELECT MURMUR_HASH('hello', 'world');
30 *
31 * Here's a way to reduce an entire table to a single order-independent hash:
32 *
33 * mysql> SELECT BIT_XOR(CAST(MURMUR_HASH(col1, col2, col3) AS UNSIGNED)) FROM

35 *
36 * Note - This code makes a few assumptions about how your machine behaves -
37 * 1. We can read a 4-byte value from any address without crashing
38 * 2. sizeof(int) == 4

40 * And it has a few limitations:
41 * 1. It will not work incrementally.
42 * 2. It will not produce the same results on little-endian and big-endian
