253 /// The maximum display width of this column.
254 ///
255 /// The "display width" is the number of code points that is needed to print
256 /// out the string representation of a value. It can be given by the user
257 /// both explicitly and implicitly. If a user creates a table with the columns
258 /// "a VARCHAR(3), b INT(3)", both columns are given an explicit display width
259 /// of 3 code points. But if a user creates a table with the columns
260 /// "a INT, b TINYINT UNSIGNED", the first column has an implicit display
261 /// width of 11 (-2147483648 is the longest value for a signed int) and the
262 /// second column has an implicit display width of 3 (255 is the longest value
263 /// for an unsigned tinyint).
264 /// This is related to storage size for some types (VARCHAR, BLOB etc), but
265 /// not for all types (an INT is four bytes regardless of the display width).
266 ///
267 /// A "code point" is basically a numeric value. For instance, ASCII
268 /// compromises of 128 code points (0x00 to 0x7F), while unicode contains way
269 /// more. In most cases a code point represents a single graphical unit (aka
270 /// grapheme), but not always. For instance, É may consists of two code points
271 /// where one is the letter E and the other one is the quotation mark above
272 /// the letter.
