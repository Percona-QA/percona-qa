1007 // because their strnxfrm function maps one byte to one byte. However, the
1008 // mapping is not injective, so the inverse function will take in an extra
1009 // index parameter containing information to disambiguate what the original
1010 // character was.

1013 // src -> (dst, idx)
1014 //

1016 // idx.

1018 // For the inverse, we have:

1022 //
