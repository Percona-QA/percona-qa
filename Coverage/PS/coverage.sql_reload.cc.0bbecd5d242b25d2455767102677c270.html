444 Implementation of FLUSH TABLES @<table_list@> WITH READ LOCK.
445
446 In brief: take exclusive locks, expel tables from the table
447 cache, reopen the tables, enter the 'LOCKED TABLES' mode,
448 downgrade the locks.
449 Note: the function is written to be called from
450 mysql_execute_command(), it is not reusable in arbitrary
451 execution context.
452
453 Required privileges
454 -------------------
455 Since the statement implicitly enters LOCK TABLES mode,
456 it requires LOCK TABLES privilege on every table.
457 But since the rest of FLUSH commands require
458 the global RELOAD_ACL, it also requires RELOAD_ACL.
459
460 Compatibility with the global read lock
461 ---------------------------------------
462 We don't wait for the GRL, since neither the
463 5.1 combination that this new statement is intended to
464 replace (LOCK TABLE @<list@> WRITE; FLUSH TABLES;),
465 nor FLUSH TABLES WITH READ LOCK do.
466 @todo This is not implemented, Dmitry disagrees.
467 Currently we wait for GRL in another connection,
468 but are compatible with a GRL in our own connection.
469
470 Behaviour under LOCK TABLES
471 ---------------------------
472 Bail out: i.e. don't perform an implicit UNLOCK TABLES.
473 This is not consistent with LOCK TABLES statement, but is
474 in line with behaviour of FLUSH TABLES WITH READ LOCK, and we
475 try to not introduce any new statements with implicit
476 semantics.
477
478 Compatibility with parallel updates
479 -----------------------------------
480 As a result, we will wait for all open transactions
481 against the tables to complete. After the lock downgrade,
482 new transactions will be able to read the tables, but not
483 write to them.

486 -------------------------------------
487 - you can't flush WITH READ LOCK a non-existent table
488 - you can't flush WITH READ LOCK under LOCK TABLES

491 ------------------------------------
492 You can only apply this command to existing base tables.
493 If a view with such name exists, ER_WRONG_OBJECT is returned.
494 If a temporary table with such name exists, it's ignored:
495 if there is a base table, it's used, otherwise ER_NO_SUCH_TABLE
496 is returned.

499 ------------------------
500 For MERGE table this statement will open and lock child tables
