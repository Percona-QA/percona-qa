522 // A backup engine for organizing and managing backups.
523 // This class is not user-extensible.
524 //
525 // This class declaration adds "Write" operations in addition to the
526 // operations from BackupEngineAppendOnlyBase and BackupEngineReadOnlyBase.
527 //
528 // # Concurrency between threads on the same BackupEngine* object
529 //
530 // As of version 6.20, BackupEngine* operations are generally thread-safe,
531 // using a read-write lock, though single-thread operation is still
532 // recommended to avoid TOCTOU bugs. Specifically, particular kinds of
533 // concurrent operations behave like this:
534 //
535 // op1\op2| Read | Append | Write
536 // -------|-------|--------|--------
537 // Read | conc | block | block
538 // Append | block | block | block
539 // Write | block | block | block
540 //
541 // conc = operations safely proceed concurrently
542 // block = one of the operations safely blocks until the other completes.
543 // There is generally no guarantee as to which completes first.
544 //
545 // StopBackup is the only operation that affects an ongoing operation.
546 //
547 // # Interleaving operations between BackupEngine* objects open on the
548 // same backup_dir
549 //
550 // It is recommended only to have one BackupEngine* object open for a given
551 // backup_dir, but it is possible to mix / interleave some operations
552 // (regardless of whether they are concurrent) with these caveats:
553 //
554 // op1\op2| Open | Read | Append | Write
555 // -------|--------|--------|--------|--------
556 // Open | conc | conc | atomic | unspec
557 // Read | conc | conc | old | unspec
558 // Append | atomic | old | unspec | unspec
559 // Write | unspec | unspec | unspec | unspec
560 //
561 // Special case: Open with destroy_old_data=true is really a Write
562 //
563 // conc = operations safely proceed, concurrently when applicable
564 // atomic = operations are effectively atomic; if a concurrent Append
565 // operation has not completed at some key point during Open, the
566 // opened BackupEngine* will never see the result of the Append op.
567 // old = Read operations do not include any state changes from other
568 // BackupEngine* objects; they return the state at their Open time.
569 // unspec = Behavior is unspecified, including possibly trashing the
570 // backup_dir, but is "memory safe" (no C++ undefined behavior)
571 //
