156 B2. All attributes QA in Q can be divided into 3 overlapping groups:

162 equivalence of DISTINCT and GROUP queries.
163 - NGA = QA - (GA union C) = {NG_1, ..., NG_m} - the ones not in
164 GROUP BY and not referenced by MIN/MAX functions.

166 B3. If Q has a GROUP BY WITH ROLLUP clause the access method is not
167 applicable.

172 SA3. The attribute C can be referenced in the WHERE clause only in
173 predicates of the forms:
174 - (C {< | <= | > | >= | =} const)
175 - (const {< | <= | > | >= | =} C)
176 - (C between const_i and const_j)
177 - C IS NULL
178 - C IS NOT NULL
179 - C != const
180 SA4. If Q has a GROUP BY clause, there are no other aggregate functions
181 except MIN and MAX. For queries with DISTINCT, aggregate functions
182 are allowed.
183 SA5. The select list in DISTINCT queries should not contain expressions.
184 SA6. Clustered index can not be used by GROUP_MIN_MAX quick select

208 - PA(G_i1,...G_iq) is an arbitrary predicate over an arbitrary subset
