33 /*
34 class Endpoint allows to define prefix ranges.
35
36 Prefix ranges are introduced below.
37
38 == Basic Ranges ==
39 Let's start from basic ranges. Key Comparator defines ordering of rowkeys.
40 Then, one can specify finite closed ranges by just providing rowkeys of their
41 endpoints:
42
43 lower_endpoint <= X <= upper_endpoint
44
45 However our goal is to provide a richer set of endpoints. Read on.
46
47 == Lexicographic ordering ==
48 A lexicographic (or dictionary) ordering satisfies these criteria: If there
49 are two keys in form
50 key_a = {prefix_a, suffix_a}
51 key_b = {prefix_b, suffix_b}
52 and
53 prefix_a < prefix_b
54 then
55 key_a < key_b.
56
57 == Prefix ranges ==
58 With lexicographic ordering, one may want to define ranges in form
59
60 "prefix is $PREFIX"
61
62 which translates to a range in form
63
64 {$PREFIX, -infinity} < X < {$PREFIX, +infinity}
65
66 where -infinity will compare less than any possible suffix, and +infinity
67 will compare as greater than any possible suffix.
68
69 class Endpoint allows to define these kind of rangtes.
70
71 == Notes ==
72 BytewiseComparator and ReverseBytewiseComparator produce lexicographic
73 ordering.
74
75 The row comparison function is able to compare key prefixes. If the data
76 domain includes keys A and B, then the comparison function is able to compare
77 equal-length prefixes:
78
79 min_len= min(byte_length(A), byte_length(B));
80 cmp(Slice(A, min_len), Slice(B, min_len)); // this call is valid
81
82 == Other options ==
83 As far as MyRocks is concerned, the alternative to prefix ranges would be to
84 support both open (non-inclusive) and closed (inclusive) range endpoints.
85 */
