138 /**
139 When storing an object, Weak_object_impl::store is called.
140
141 The primary key for the object is created, returning a new Primary_id_key
142 instance with the object id, given by Entity_object::id(). The primary
143 key instance represents the primary key value.
144
145 Then, a Raw_record is created by Raw_table::prepare_record_for_update.
146 Here, the record is essentially retrieved by find_record based on the
147 primary key. This is done by creating a raw access key, which is the primary
148 key value with a physical representation suitable for looking up in a storage
149 engine. The raw key is used to get the appropriate record from the dd tables
150 by calling ha_index_read_idx_map. By instrumenting this function to return
151 1, indicating no record found, we will provoke insert.
152
153 Then, for insert, Weak_object_impl::store will first call
154 prepare_record_for_insert, which just creates a new Raw_new_record instance.
155 Then, the schema object members are copied into the appropriate table fields
156 by calling the function Schema_impl::store_attributes. This, in turn, invokes
157 the various Field*::store functions, which are set up to invoke the
158 corresponding fake_store methods, and finally, Raw_new_record::insert
159 is called, which calls ha_write_row. The write_row function in the handler
160 api is instrumented to return 0 (success).
161
162 After storing the schema object, we verify that the values saved in
163 the various fields by fake_store are the same as we used to create the schema
164 object.
165 */
