## 7. Recommendations for the team

1. **Adopt the new patterns.** New tests added since the baseline use:
   - Polling-with-deadline instead of `time.sleep`
   - On-disk byte checks (encrypted-content / plaintext-leak markers)
   - Server-log scans for error phrases (decryption errors etc.)
   - Skip-on-missing-function probes for forward compatibility
   These should be the template for future tests.

2. **Adopt `--strict-markers` permanently.** All new markers (`waldump`)
   are registered in `pyproject.toml`. A typo in a marker now fails at
   collection, not at runtime.

3. **Keep `coverage_reports/` versioned.** Each report is dated so
   week-over-week deltas are visible. The skill `/tde-coverage report`
   regenerates this file with a fresh date.
