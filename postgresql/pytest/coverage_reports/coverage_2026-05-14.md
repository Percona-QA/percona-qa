## 2. What changed today — file-by-file

### 2.1 New module — `tests/test_cipher.py` (12 SMGR-reuse tests + 6 GUC tests moved in from `test_encryption.py`)

> **Note (post-consolidation):** this file was originally created as `test_smgr_cipher_reuse.py`. After the SMGR-context-reuse tests landed, `TestTdeCipher` (the 6 `pg_tde.cipher` GUC tests previously in `test_encryption.py`) was moved here and the file was renamed to `test_cipher.py` so all cipher coverage — GUC contract + SMGR-context contract — sits in one module. The 12 tests below remain unchanged.

**Purpose:** Regression coverage for PR #554 / PG-2278. The pg_tde SMGR layer reuses OpenSSL `EVP_CIPHER_CTX` objects on read/write paths. If that reuse leaks key/IV/padding state between relations or between read and write paths, ciphertext is silently corrupted and only surfaces when:

- pages are read off disk (not from shared buffers), or
- a partitioned / CTAS / TRUNCATE / REINDEX / CLUSTER cycle re-encrypts onto a fresh `relfilenode` under the same process, or
- the server restarts and re-loads pg_tde / re-creates the contexts.

All tests use a **database-scope** key provider (matching the bash script) and `shared_buffers = 2 MB` so bulk seqscans actually go through pg_tde SMGR decrypt off disk.

| Class / Test | What it pins down |
|---|---|
| `TestSmgrCipherReuseFullSuite::test_full_suite_data_intact_across_restart` [aes_128] | End-to-end mirror of the bash `run_cipher_suite` flow on AES-128: two heaps + extended scenarios + bulk I/O + COPY → restart → verify aggregates match before/after |
| `TestSmgrCipherReuseFullSuite::test_full_suite_data_intact_across_restart` [aes_256] | Same suite parametrized on AES-256 (different context size → catches alignment regressions specific to one key length) |
| `TestSmgrCipherReuseFocused::test_two_relations_different_keys_decrypt_independently` | Two `tde_heap` relations sharing the SMGR context but with distinct per-relation keys decrypt to their own deterministic plaintext |
| `…::test_partitioned_tde_heap_one_query_tree` | Three encrypted child partitions touched in one SELECT — SMGR decrypt with three different per-relation keys in one query tree |
| `…::test_toast_external_storage_decrypts` | TOAST relation for a `tde_heap` heap is also encrypted; row payloads > 8 kB round-trip |
| `…::test_ctas_with_savepoint_rollback_preserves_pre_rollback_state` | CTAS → partial UPDATE → ROLLBACK TO SAVEPOINT — pre-rollback rows decryptable, no aborted rows leak |
| `…::test_truncate_and_reload_uses_fresh_relfilenode_correctly` | TRUNCATE allocates a fresh `relfilenode`; subsequent INSERT encrypts onto the new file with the same SMGR context, survives a restart |
| `…::test_reindex_and_cluster_decrypt_after_physical_reorder` | REINDEX rewrites the index, CLUSTER rewrites the heap — both produce decryptable storage |
| `…::test_copy_from_file_into_tde_heap_round_trips` | Server-side `COPY ... FROM '<file>'` into a `tde_heap` table; every row round-trips |
| `…::test_cold_seqscan_forces_smgr_read_off_disk` | `pg_statio_user_tables.heap_blks_read > 0` after a CHECKPOINT + cold seqscan — proves the SMGR decrypt path was exercised (not just shared buffers). PG14+ only |
| `…::test_multi_relation_interleaved_scan_completes_correctly` | UNION ALL fans across 6 encrypted heaps in one SELECT — executor round-robins SMGR reads, reused context keeps keys/IVs straight |
| `…::test_index_only_path_decrypts_heap_lookups` | `enable_seqscan = off` → btree-driven path; heap pages still go through SMGR decrypt for visibility check |

### 2.2 New module — `tests/test_unlogged_recovery.py` (7 tests)

**Purpose:** Port of `pg_tde_unlogged_test.sh` (= upstream `recovery/014_unlogged_reinit.pl`). The crash-recovery contract for UNLOGGED relations under pg_tde:

1. `_init` fork **kept**.
2. Main (`''`) fork **re-created** from the init fork during recovery.
3. `_vm` and `_fsm` forks **removed** during recovery (rebuilt lazily on demand).

PG-1805 in `test_bug_reproduction.py` covers the narrower identity-column trigger only; this module covers the broader contract on tables, sequences, and tablespace variants.

| Class / Test | What it pins down |
|---|---|
| `TestUnloggedReinitDefaultTablespace::test_unlogged_table_forks_present_before_crash` | `_init` and main forks exist immediately after CREATE UNLOGGED |
| `…::test_unlogged_sequence_nextval_before_crash` | Baseline: `nextval` on a fresh UNLOGGED sequence returns 1, 2 |
| `…::test_unlogged_main_fork_recopied_vm_fsm_removed_after_crash` | The headline contract — main re-created, VM/FSM removed, post-recovery INSERT succeeds |
| `…::test_unlogged_sequence_resets_to_one_after_crash_reinit` | UNLOGGED sequences reset on crash recovery; post-recovery `nextval` returns 1 again |
| `TestUnloggedReinitCustomTablespace::test_unlogged_table_in_custom_tablespace_reinitialised_after_crash` | Same contract on a user tablespace — proves the recovery code walks every tablespace, not just default |
| `TestUnloggedReinitSequencesAndIdentity::test_altered_sequence_and_identity_table_after_recovery` | `ALTER SEQUENCE … INCREMENT 2` + IDENTITY column survive reinit; custom increment preserved (lives in pg_sequence), value reset |

## 6. Test-design highlights

The new tests follow the patterns established in 2026-05-12 / 2026-05-13 and add two more worth adopting:

1. **Cipher parametrization on suite-level tests, not per-scenario.** The bash script runs its entire 13-step workload twice (AES-128, AES-256). The pytest port parametrizes the comprehensive end-to-end test on `cipher` but keeps focused per-scenario tests at the default cipher only. This balances correctness coverage (cross-cipher) with test-failure granularity (focused). See `TestSmgrCipherReuseFullSuite` and `TestSmgrCipherReuseFocused`.

2. **Auto-skip on missing external tooling, with fallback paths.** `TestTdeRewindSysbenchLoop.test_rewind_sysbench_driven_failover_loop` checks both `shutil.which("sysbench")` AND walks three standard paths for `oltp_insert.lua`, skipping cleanly if neither is found rather than failing in an unhelpful way. This matches how the bash test_runner.sh behaves and avoids CI flakiness on systems without sysbench's data files.

3. **Scope-preservation assertions for upgrades.** `test_pg_tde_upgrade_explicit_global_key_provider_migration` asserts the upgraded cluster has the provider in the **global** catalog AND NOT in the database catalog. Provider-scope downgrades are a silent class of regression (queries keep working but `inherit_global_providers` / CREATE DATABASE inheritance breaks). The bash scenario did not check this; the pytest port does.
